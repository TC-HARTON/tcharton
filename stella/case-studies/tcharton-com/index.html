tcharton.com — 沼津起点の自己実証体 第 1 号

旧 C/65 → 🏆 S/90 (HARTON S-Class) / 7 commit + 住所完全公開で達成

2026-04-27 取得記録 ／ 最終更新 2026-05-03 ／ 運営: T.C.HARTON

HARTON Stella は機械検証で WEB 品質を測る独立認定機関である。本機関の中核原則は「機械検証が示す、AI 時代の WEB 品質。」であり、運営主体 T.C.HARTON は dogfooding 倫理 (自社評価基準を自社サイトに先に適用する) に基づき、自社サイト tcharton.com を scanner.py 機械検証の最初の評価対象とした。本ページはその「課題 → 改善 → 取得」全行程を verbatim 公開する。

「自分が達成できない基準で他者を測ることはしない」 — HARTON Stella 信頼根拠の核

3 段ストーリー: 課題 → 改善 → 取得

1課題

初期スコア C/65 — 機械検証で発覚した品質ギャップ

2026 年 Phase 1 着手前、自社サイト tcharton.com を scanner.py で初回評価したところ、総合 65 点 (C 区分) という結果を得た。本機関の認定基準は ★ HARTON Stella が 70 点 + 致命的 NG 0 件 であり、自社サイト自体が認定基準未達という事実を直視する必要があった。

具体的な課題:

  • A 軸 基礎: HSTS preload 未登録 / Core Web Vitals 一部未達 (LCP > 2.5s)
  • B 軸 防御: CSP 未配信 / Trusted Types 未適用 / WP 管理面検出
  • C 軸 AI 検索: JSON-LD @type 単一 (配列化前) / sameAs Wikidata 未連携 / GBP CID 未取得
  • D 軸 経営: NAP 不完全 (住所 △ / streetAddress 非公開) / OGP 未整備

これらは scanner ④ S クラス最高技術責任者の TCHARTON-AUDIT で全件機械検出された。「自分が達成できない基準で他者を測らない」原則に従い、認定機関として運用開始する前に自社サイトを認定基準まで引き上げる必要があった。

2改善

7 commit + 住所完全公開 — 90 日相当の機械検証ベース改善

scanner.py の検出結果を逐次解消する形で、7 段階の commit と 1 個の重要決定 (住所完全公開) で改善した。各 commit は SPEC v3.4 / GOOGLE-STANDARDS v2.0 / GEO-STANDARDS v2.1 整合の機械検証可能な変更のみで構成される (人手判断は介在しない)。

3取得

🏆 S/90 (HARTON S-Class) 取得 — 自己実証体 第 1 号確定

commit 36d4328 で住所完全公開した直後の scanner.py 再判定で 総合 90 点 / 必須条件 4/4 + 1 保留 / 致命的 NG 0 件 / NAP 100 点 を達成し、★★★ HARTON Stella S-Class 認定を取得した。

取得詳細:

  • 総合スコア: 90 点 (★★★ 認定基準 90 点をジャストで達成 / dogfooding 倫理上「自社の集客のため」ではないため過剰追求せず)
  • 必須条件: 4/4 達成 + 1 保留 (CR-3 Cloudflare Turnstile は別実装途中)
  • 致命的 NG: 0 件 (HTTPS / SSL / WP 管理面 / CMS バージョン 全件 clean)
  • NAP 完全一致: 100 点 (Schema.org PostalAddress + GBP + footer 三重整合)
  • 取得 commit: 36d4328 (2026-04-27)

これにより、tcharton.com は HARTON Stella の 自己実証体 第 1 号 (the first dogfooding subject) として確定した。

dogfooding 倫理の証明 — なぜ自己実証体が必要か

世界中のどの認定機関とも異なり、HARTON Stella の評価ロジックは scanner.py という独自開発の機械検証エンジンで、4 軸並列独立評価 + 必須 5 条件 + 致命的 NG 4 項目で構成される。この基準を、まず自社サイト tcharton.com で実証し、★★★ を取得した。

  • L1 自己適用: 自社サイト tcharton.com が認定基準で評価される (scanner.py 月次再判定対象)
  • L2 自己拘束: 評価結果は自己例外なし (致命的 NG / 降格条件は他事業者と同条件)
  • L3 自己公開: 機械検証ログを全件 verbatim 公開する (本ページがその物理証拠)

この原則 (本ページ冒頭に blockquote 引用) が、本ページで verbatim 公開する 7 commit timeline の核にある。スコア向上のため自社の住所を完全公開する判断も、評価機関としての中立性を維持するための dogfooding 倫理から導かれた。

未来事業者へ — あなたのサイトが ★ を取得するための具体経路

tcharton.com の経緯は特殊事例ではない。同じ機械検証基準と同じ 5 Step (90 日) で、全国の事業者が ★ HARTON Stella ↦ ★★ HARTON Stella 優良 ↦ ★★★ HARTON Stella S-Class まで到達可能である。

  1. Step 1 (HTTPS 常時 SSL 化 + HSTS preload): tcharton.com では commit #1 で対応 → A 軸 +15 点
  2. Step 2 (JSON-LD Schema.org + GBP): commit #2-#4 で対応 → C 軸 +12 点
  3. Step 3 (CSP + Trusted Types): commit #1 で対応 → B 軸 +10 点 / 致命的 NG 解消
  4. Step 4 (GEO/LLMO 最適化): llms.txt + FAQPage + 公的リンク 5+ → C 軸 +10 点
  5. Step 5 (Core Web Vitals + WCAG 2.2 AA): SSG + 画像最適化 → A 軸 +8 点 / D 軸 +10 点

各 Step の具体的 config 値・header 文字列・ファイルパスは 改善ガイダンスで公開する。自社サイトの ★ 取得を目指す事業者は 掲載申請 (無料)から開始可能である。
