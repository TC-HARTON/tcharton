よくあるご質問

WordPress でも改修できますか？

対応可能です。ただし WordPress の構造的脆弱性（管理画面公開 / プラグイン依存 / DB 攻撃面）は完全解消できません。★★★ 取得には静的サイトへの移行を推奨する場合があります（移行費用は別途見積）。

必ず ★★★ が取れますか？

事前 audit レポートで ★★★ 取得の可否を判定します。技術的に ★★★ が困難なサイト（極度に古い CMS / 独自フレームワーク等）は事前にお伝えし、★★ までのリフトプランをご提案します。

サイトを停止せずに改修できますか？

はい。Cloudflare Workers Static Assets での段階リリース or Cloudflare Pages preview deploy を使い、本番稼働を維持しつつ改修内容を逐次切替可能です。

月額保守は必ず必要ですか？

★★★ 認定の維持には必須です（dogfooding 倫理）。OWASP CVE / Google AI Overviews 仕様 / Core Web Vitals 閾値は毎月変動するため、保守なしで構造的に維持不可能です。改修のみで月額契約しない選択も可能ですが、認定は完了時点限定となります。

audit を経ずに直接改修依頼できますか？

原則 audit を先行します。改修範囲・工数・★★★ 取得可否を事前に明示しないと、見積精度が低下するためです。例外的に既存 ★ 認定済サイトの再改修は audit を省略可能（その場合も簡易事前確認を実施）。
