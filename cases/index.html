公開済み事例（1 件）

以下は 課題 → 思考プロセス → 解決 の 3 段構成で記述。「何をしたか」だけでなく「なぜそう判断したか」を言語化することで、再現性のある制作プロセスとして公開します。

Local National ★★★ 自己実証体 自社サイト WEB構築 保守運用 個人事業主

tcharton.com 構築 — 「自分が達成できない基準で、他者を測ることはしない」

2026-04 公開 / 個人事業主 / 期間 約 3 週間 / 制作 大内 達也（単独）

Stage 1 / Challenge

課題: 認定機関を設計するなら、自社が真っ先に基準を満たす必要があった

T.C.HARTON は 4 つの事業（WEB構築 / 保守運用 / AI予測 / HARTON Certified 認定）を提供する事業者であり、なかでも HARTON Certified（ certification.tcharton.com）は他社の WEB サイトを ★ / ★★ / ★★★ で評価する 第三者認定機関。ここで根本的な問いに直面した:

「他社を ★★★ で評価する以上、自社サイトが ★★★ を取得していなければ倫理的に矛盾する。しかも、評価基準は AI 検索時代を踏まえた次世代水準であり、世の WEB 制作の中央値からは遠い位置にある。」

さらに静岡県 5 都市 902 件の業界実測（2026-05 自社調査）では、業界中央値スコアは 24 点（100 点満点）。当社が設計した HARTON Certified ★★★ の閾値は 90 点 + 致命的 NG 0 件 + 必須条件 5 件全件達成。業界中央値の 3.8 倍の水準を、自社で先行達成する必要があった。

Stage 2 / Thinking Process

思考プロセス: 4 つの判断と、それぞれの「なぜ」

判断 1: なぜ静的サイトか — 「動的機能 = 攻撃面」と捉えた

一般的な選択肢は WordPress（CMS 市場で約 4 割の業界シェア / W3Techs 公開統計）。だが当社が静岡県 902 件を機械検証した結果（参考値: 32.5% で致命的脆弱性 = 管理画面公開等を検出）、「公開し続ける限りパッチを当て続ける運用前提」は、HARTON Certified の「致命的 NG ゼロを構造的に保証する」哲学と矛盾する。そもそも管理画面を持たない＝攻撃面（=ハッキング経路）そのものを構造的に消す = 静的サイト + Cloudflare Workers Static Assets を選択。

判断 2: なぜ自社製 spec-checker.js か — 「規範と検証は同一エンジニアが書くべき」

Lighthouse / WAVE / pa11y 等の汎用ツールは存在する。だが当社の評価基準（SPEC v3.5 + GOOGLE-STANDARDS v2.0 + GEO-STANDARDS v2.1）は、汎用ツールが扱う共通項を超え、独自の 2,554 検証項目に体系化されている。規範を書く担当が、その検証エンジンも自分で書くことで「機械検証可能でない規範は書かない」という強制力が働く。これは dogfooding の最深層適用。

判断 3: なぜ pre-push hook で S-RANK ゲートを敷くか — 「人間の判断を信用しない」

どれほど厳格な規範を書いても、実装時の「ちょっと急ぎだから後で直す」「軽微だから例外」が積み上がるのが現場の実態。これを構造的に排除するため、git の push 直前に走るスクリプト（.git/hooks/pre-push）で機械検証エンジンを強制実行し、FAIL > 0 または合格率 < 100% で 物理的に push を拒否。これは CSP（= ブラウザがどのスクリプトを許可するかを決めるセキュリティ設定）の規範違反解消事案（2026-05-03）でも作動 — 危険なインラインスクリプトが混入した瞬間 FAIL し、本番デプロイ前に発見できた。

判断 4: なぜ AI 検索 (GEO) を最優先軸に置いたか — 「Google だけでは負ける時代」

ChatGPT / Perplexity / Gemini が「直接回答」を返す時代、ユーザーは Google 検索結果ページを経由しなくなる。 Aggarwal et al. (KDD 2024) によれば、AI に引用される確率は「Quotation + Statistics + Cite Sources」3 戦略の組み合わせで 最大 +40.6%向上。当社は GEO 9 戦略のうち White Hat 7 種を全ページに適用 — これは SEO の延長ではなく、「AI に正確に引用される」ための情報設計として再定義したもの。

Stage 3 / Resolution

解決: HARTON Certified ★★★ を、自社が業界に先駆けて取得

結果、当社は HARTON Certified ★★★ を自社で取得した最初の制作事業者となり、Stage 1 で立てた「自分が達成できない基準で他者を測ることはしない」を物理的に証明。以下の数値はすべて第三者機関の API / 自社製機械検証エンジンの出力で再現可能な値のみ掲載しており、「自社測定」「特定期間」を併記し一般化された主張を避けています（SPEC §0.0 H-2 準拠）。

事例公開ポリシー

すべての事例は以下の原則で公開します。

  1. ① 許諾済みのみ

    公開前にお客様から書面または明確な許諾を取得。匿名化希望にも対応（業種 + 地域のみ表示）。

  2. ② 検証可能な数値のみ

    「○% 削減」「□倍増加」等の数値はすべて測定方法・期間・基準値を併記。再現性のない数値は掲載しない。

  3. ③ Scope-Explicit

    「自社測定」「特定期間」「特定 KPI 限定」等の境界を必ず明記。一般化された主張は避ける（SPEC §0.0 H-2 準拠）。

  4. ④ 他社中傷の禁止

    前任業者・競合の批判は掲載しない。「○○ができていなかった」ではなく「○○を実装した」の能動形で記述。

  5. ⑤ 削除権

    お客様からの削除・修正依頼にはいつでも対応。特に M&A・事業譲渡時は速やかに対応します。
