自社サイト WEB構築 保守運用 個人事業主

tcharton.com 構築（2026-04 公開）

課題

T.C.HARTON 事業立上げに伴う公式サイト新規構築。SPEC v3.2 / 2,554 検証項目への完全準拠が要件。事業内容（旧 harton/site-builder → 新 3 本柱: WEB構築 / 保守運用 / AI予測）の刷新を反映。

施策

  • ▶SPEC v3.2 完全準拠の 18 ページ静的サイト構築
  • ▶spec-checker.js（自社製・2,554 検証項目）+ pre-push hook で S-RANK ゲート
  • ▶Body Theme Variants（marketing / reading）使い分け
  • ▶無料診断シミュレーター実装（5 質問 × 3 軸スコア）
  • ▶GEO 9 戦略のうち White Hat 7 種を全ページ適用

効果（2026-04-25 時点）

  • ▶spec-checker FAIL = 0 / 合格率 100% を全 18 ページで達成
  • ▶dist/output.css 14.9 KB（目標 40 KB 以下を大幅クリア）
  • ▶JSON-LD 5 種完備（ProfessionalService / WebSite / FAQPage / BreadcrumbList / Person）
  • ▶本番公開後の Lighthouse / CWV / SSL Labs / Mozilla Observatory スコアは公開時に追記予定

技術スタック

HTML Living Standard / Tailwind CSS 3.4 / Cloudflare Pages / Node.js（spec-checker.js, sync-spec.js, verify-all.js）

事例公開ポリシー

すべての事例は以下の原則で公開します。

  1. ① 許諾済みのみ

    公開前にお客様から書面または明確な許諾を取得。匿名化希望にも対応（業種 + 地域のみ表示）。

  2. ② 検証可能な数値のみ

    「○% 削減」「□倍増加」等の数値はすべて測定方法・期間・基準値を併記。再現性のない数値は掲載しない。

  3. ③ Scope-Explicit

    「自社測定」「特定期間」「特定 KPI 限定」等の境界を必ず明記。一般化された主張は避ける（SPEC §0.0 H-2 準拠）。

  4. ④ 他社中傷の禁止

    前任業者・競合の批判は掲載しない。「○○ができていなかった」ではなく「○○を実装した」の能動形で記述。

  5. ⑤ 削除権

    お客様からの削除・修正依頼にはいつでも対応。特に M&A・事業譲渡時は速やかに対応します。
