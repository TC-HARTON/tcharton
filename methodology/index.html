評価方法論と品質根拠

T.C.HARTON が WEB 品質を機械保証する根拠を完全公開します。spec-checker.js 2,554 検証項目 / 5 評価軸 / 公式 8 一次ソース—すべて再現可能な手順として開示します。

2026年5月3日 / 最終更新 2026年5月3日

5 評価軸の根拠と一次ソース

すべての WEB 構築は以下 5 軸で機械検証します。各軸は独立評価され、項目別の事実減点で総合スコアを算定します（4 軸並列独立評価フレーム + CV 補完軸 / SPEC §0.0.10 厳格化原則準拠）。

①

技術

— Lighthouse 95+ / Core Web Vitals

LCP < 2.5s / INP < 200ms / CLS < 0.1 を pre-push hook で機械検証。web.dev / Core Web Vitals（Google 公式）の基準値を直接参照。

②

コンテンツ

— 一次情報・E-E-A-T・Lead Evidence Block

Google 有用なコンテンツ準拠。記事冒頭に Lead Evidence Block（最初の `<h2>` より前に数値・公的リンク・引用句のいずれか）を必須配置（SPEC §4.13）。

③

信頼

— SSL Labs A+ / OWASP / IPA

OWASP Top 10:2025 + IPA 安全なウェブサイトの作り方 改訂第7版 準拠。代表プロフィール完全公開・料金透明・所在地明示で E-E-A-T を担保。

④

LLMO（AI 検索最適化）

— GEO 9 戦略 White Hat 7 種

arXiv:2311.09735（KDD 2024）の 9 戦略のうち White Hat 7 種（Quotation / Statistics / Fluency / Cite Sources / Technical Terms / Easy-to-Understand / Authoritative）を必須実装。Keyword Stuffing は SEO ペナルティリスクのため採用禁止。

⑤

CV（コンバージョン）

— CTA 配置 / フォーム最適化 / GA4 計測

Google SEO スターターガイド準拠。CTA 最適配置 / 2 分送信フォーム / 3 経路（メール・電話・LINE）/ GA4 イベント計測。

GEO 最強コンビ — Quotation + Statistics + Cite Sources

Aggarwal et al. (KDD 2024) が示した 9 戦略のうち、G-1 Quotation（+27.8%）+ G-2 Statistics（+25.9%）+ G-4 Cite Sources（+24.9%）の 3 種を同一段落で組合わせると相乗効果が確認されています。HARTON はこれを「最強コンビ」として全ページ Lead Evidence に必須実装します。
