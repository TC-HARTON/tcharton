### 3.2 カテゴリ別所見

#### (A) ヒーローインパクト 11/20
- 強み: 「『誰も来ない HP』を、終わらせる。」は痛点直撃でローカル競合を凌駕（L1-L5 のフラットな「格安・高品質」訴求より明確に差別化）。
- 弱み: 「高品質を、静岡から」だけでは Linear の「**purpose-built tool**」や Stripe の「**Financial Infrastructure**」級の **カテゴリ定義** に到達していない。「S クラス保証」は造語で外部認知ゼロ → 説明コスト過大。
- **失点根拠:** ヒーロー左に SVG アニメ（Old → 粒子 → New）を配しているが、5 秒シーケンスはモバイルで CLS リスクと注意分散を招く。Garden Eight / mount.jp は 1 枚絵で完結。

#### (B) デザイン洗練度 12/20
- 強み: ディープティール #1B4965 + ゴールドアクセント #D4AF37 のパレットは静岡ローカル群より一段上。
- 弱み: **アワード受賞ゼロ**（C2/C3/C4 は Awwwards・CSS DA を毎年受賞）。タイポグラフィは Inter + Noto Sans JP のテンプレ的組み合わせ、独自書体・カーニング調整なし。背景の dot grid + radial-gradient は 2020 年代前半のトレンド表現で、海外ベンチでは既に陳腐化。

#### (C) 機能の差別化 11/15
- **AI 予測モデル開発 + WEB 制作** の併売は国内同業で稀少（ベイジ・LIG はマーケ寄り、ニジボックスは UX 寄り、AI 予測まで踏み込む WEB 制作会社はほぼ存在しない）。
- 「保守込み制作」自体はちょこ WEB / locoprio 等の月額型と被るが、tcharton は「3.8 倍品質」のエビデンス付きで差別化。
- **失点根拠:** AI 予測の事例・実装スタック（XGBoost? LSTM? Prophet?）が TOP に一切無く、抽象的なまま。

#### (D) クライアント実績訴求 3/15 ★最大失点
- LIG は「制作実績『LIG WORKS』」を専用サイト化、ベイジは「年 400 件リード」を数値化、Garden Eight は Awwwards 受賞作の動画表示。
- tcharton TOP には **クライアントロゴ・実績スクショ・受賞バッジが一切ない**。あるのは「902 社調査」（自社調査であって他社実績ではない）と「1 号客限定キャンペーン（残り 2 枠）」 → これは **実績ゼロを暗に告白** している。
- **致命的:** 100 点到達には「実績ゼロ」をどう逆手に取るかの戦略が必須。

#### (E) ナラティブ・ストーリー 7/10
- 「私たちの想い」「代表 大内 達也 直接対応」「沼津拠点」など個人事業主の人間味は出ている（dewman の「複雑な承認フロー無し」と同系統）。
- 不足: Stripe・Linear のような **カテゴリ宣言型ナラティブ**（「インターネットの GDP を伸ばす」）に相当する大きな物語が欠如。「S クラス保証」は手法名であって、世界観ではない。

#### (F) UX 完成度 7/10
- 強み: a11y skip link、reduced-motion 対応、24×24 タップターゲット、モバイルメニューの ARIA dialog 準拠。
- 弱み: ヒーロー右カラムの情報密度過多（H1 + sub + evidence + 3 cards + promo + CTA + meta が 1 画面に圧縮）→ Linear の「H1 + sub + 1 CTA」のミニマリズムに比べ視線誘導が散漫。

#### (G) 技術的優位 9/10
- 国内同業では **最高クラス**。Trusted Types + 厳格 CSP + HSTS preload + 構造化データ 4 種 + spec-checker 2554 項目自動検証は、ベイジ・LIG・Garden Eight ですら未実装の領域。Vercel/Stripe レベル。
- 失点 1 点は **静的 HTML 直書き**（フレームワーク非採用）の運用持続性懸念。

## 6. 100 点到達の競合超越戦略（5 本柱）

### 戦略 1: 「実績ゼロ」を **証跡型ナラティブ** で逆転（+15 点見込）
- **削除:** 「1 号客限定キャンペーン（残り 2 枠）」プロモは実績ゼロを露呈しているので **撤廃または別ページに退避**。
- **新設:** 「**実績ではなく、検証で語る**」セクション
  - spec-checker 2554 項目の **PASS スコアバッジ**（自社サイト現スコアを可視化）
  - 「902 社中 S クラスは 0 社、本サイトのみ」という比較グラフ
  - Lighthouse / PageSpeed Insights の実測値カード（100/100/100/100 表示）
  - 「自社サイトを唯一の実績とする」開示型コピー（透明性で逆転）

### 戦略 2: ヒーローを **カテゴリ定義型** に再構築（+5 点）
- 現: 「『誰も来ない HP』を、終わらせる。」
- 案 A: 「**S クラス WEB を、静岡から定義する。**」 / sub: 「Google・WCAG・OWASP・GEO の 4 法規を 100% 満たす唯一の制作基準。」
- 案 B（Stripe 型）: 「**WEB 品質のインフラを、中小企業に。**」
- 推奨: **案 A**（「S クラス」を造語ではなく「定義者」のポジションに昇格させる）。
- **SVG アニメは下方スクロールに移動**、ヒーローは Linear 流の H1 + sub + 1 CTA + ロゴ（自社受賞バッジ）構成へ簡素化。

### 戦略 3: **自社認証バッジ制度（HARTON Certified）連携**（+8 点）
- certification セッション⑤の HARTON Certified MASTER-PLAN を TOP に直結。
- TOP に「HARTON Certified ★★★ S クラス取得サイト一覧」セクションを設置 → 認定先サイトを **疑似クライアント実績** として陳列（自社で認証発行 → 自社で実績化、というメタ戦略）。
- これによりベイジ「年 400 件リード」級の数値訴求に並ぶ。

### 戦略 4: **アワード応募のロードマップ宣言**（+4 点）
- TOP フッター近傍に「**Awwwards / CSSDA 2026 Q3 応募予定**」を明示。
- 受賞前から「目指している」ことを宣言することで C2/C3/C4 と同じ土俵にいる印象を演出。
- 同時に **HARTON Certified を Awwwards Honorable Mention 級デザインで構築** → 受賞確度を上げる。

### 戦略 5: **ヒーロー右カラム情報削減**（+3 点）
- 現: H1 + sub + evidence + 3 cards + promo + CTA + meta（7 要素）
- 改: H1 + sub + 1 CTA + 数値証跡 1 行（4 要素）
- 3 カードは下方の「Services」セクションに分離。プロモは撤廃（戦略 1）。CTA 直下の「申込から 1 営業日以内」microcopy は維持。
- ヒーロー左の 5 秒 SVG アニメは **3 秒・自動停止**にトリミング（CLS リスク低減 + 集中度向上）。

## 9. 結論

- **現状 60/100** — 国内ローカル群（46）には圧倒的優位、国内中堅（79〜87）には デザイン・実績で劣後、海外ベンチ（93）には ナラティブとアワードで劣後。
