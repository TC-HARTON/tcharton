# 世界トップ WEB 制作 / 表彰サイト 5 社 深掘り分析 vs tcharton.com

**作成日:** 2026-05-08
**対象:** cssdesignawards.com / thefwa.com / awwwards.com / garden-eight.com / mount.jp
**比較基準:** tcharton.com（公開 1 か月 / 週 32 user / CV 0 / 沼津拠点個人事業）
**調査手法:** 全サイト WebFetch 実行（FWA はトップが Cloudflare/JS 制限のため /awards で補完）

---

## 1. エグゼクティブサマリ（300 字）

5 サイトに共通するのは「**写真・実績・名前**で語り、言葉で語らない」という鉄則である。Awwwards / CSSDA は他社の受賞実績をギャラリー化し、garden-eight / mount.jp はクライアント名を即座に列挙、FWA は受賞作品ビジュアルだけで成立している。一方 tcharton.com は **「HARTON Stella ★★★」「機械検証」「GEO 9 戦略」「SPEC v3.7」という自社造語と学術用語**でヒーローを埋め尽くし、実績画像も具体クライアント名も無い。トップ 5 は **0.5 秒で「凄そう」「依頼したい」と感情を喚起**するが、tcharton は **読解 30 秒以上を要求し、しかも読み終えても「自社評価の自社認定」と気付いて離脱**する。決定的欠陥は造語密度・実績不在・ビジュアル衝撃ゼロの 3 点。

## 2. 5 サイト個別分析

### 2.1 cssdesignawards.com（CSSDA）

**A. ファーストビュー**: 「**AWARDED 2026 MAY 8**」+ Featured: 「**PieterKoopt® shows there's another way.**」+「The art world can feel distant, slow, and unnecessarily complex.」
極端な短文。サイトの説明文は **0 行**。代わりに「今日表彰した作品」を巨大カードで提示する。

**B. ナビゲーション**: Home / Nominees / Winners / About / Judges / Blog / Contact / SUBMIT / Login（9 項目だが `SUBMIT` と `Login` を視覚的に分離、機能ナビは実質 6）。

**C. 「何をする会社か」**: 一切説明しない。**作品が説明である**。About を踏まないと出てこない。

**D. 実績の見せ方**: 6 名の審査員が UI / UX / Innovation の 3 項目を **0-10 スコア**で評価し集計。Newest Nominees 3 件 + Previous WOTDs 9 件をグリッド掲載。**「他者が他者を評価する」**ため信頼が成立。

**E. インタラクション**: 投票（Vote）/ 閲覧（View）に絞った 2 系統 CTA。

**F-G. タイポ/カラー**: 白背景 + ネイビー + チャコール。極めて抑制的。作品サムネのカラフルさを引き立てる役割に徹する。

**H-I. 余白/技術**: 余白量「中」、グリッド構造「強」。ヘッダーに framework の痕跡乏しく、custom CMS 推測。

**J. CTA**: 一般訪問者 → Vote / View、サイト制作者 → SUBMIT。**動線が 2 本に純化**。

> **学び:** 評価制度を売るなら、評価者を**外部化**し、しかも**他者作品を評価し続けて見せる**。tcharton は自分で自分を評価して終わっている。

### 2.2 thefwa.com（FWA）

トップは Cloudflare + JS 重画面で WebFetch から本文抽出不可（タイトル「Awards - The FWA」のみ取得）。/awards も同様。これ自体が示唆的で、**FWA は「コンテンツ」ではなく「体験」がトップページである**ことを意味する。WebGL / Canvas / 全画面ビデオで構成され、テキスト DOM が極小。

**学べる点（公知情報による補足）**:
- ヒーローは **動画・WebGL の作品サムネ巨大グリッド** が主役。コピーは皆無に等しい。
- ナビは Awards / FOTD / Submit / Hall of Fame など 5 程度に圧縮。
- カーソルは独自カスタム（円形フォロワー）。スクロールで作品が次々ロード。
- 「**何をする会社か**」を **言葉でなくビジュアルで証明**する究極形。

> **学び:** 言葉を増やす方向ではなく、**1 枚の作品サムネで黙らせる**方向。tcharton はこの逆を行っている。

### 2.3 awwwards.com

**A. ファーストビュー**: 「**Site of the Day May 8, 2026 / Score 7.29 of 10 / Studio Namma**」「**Visit Sotd.**」
日次更新の「今日のサイト」とスコアを最上部に固定。

**B. ナビゲーション**: Awards / Trending / Categories / Technology / Collections / Blog / Directory / Academy / Jobs / Market / Submit Website / Be Pro。一見多いが、すべて **「作品を別軸で切り直す」インデックス**であり、自社サービス説明は 0 項目。

**C. 「何をする会社か」**: トップに「**Awwwards - Website Awards - Best Web Design Trends**」のサブタイトル相当のみ。残りはすべて作品。

**D. 実績の見せ方**: SOTD 6 + Nominees 6 をカード化。各カードに **サムネ / カテゴリ / プロジェクト名 / 実 URL / 制作者プロフィール画像 / PRO バッジ**。「Vote Now」が並ぶ。

**E. CTA**: Be Pro / Submit Website / Vote Now / View ○○ の 4 系統。各セクション末に「View Nominees / View Winners / View Academy / View Collections / View Directory / View Market」と **同一文法で並べ視認性を最大化**。

**F-G**: 白基調 + 黒文字 + 強アクセント色は作品サムネに任せる。

**H. 技術ヒント**: `/assets/awwwards.com/`、`/awards/media/cache/thumb_user_70/` から Django + 自前 CDN 推測。**敢えて派手な WebGL を入れず**、速度を最重視（皮肉なことに awwwards 受賞サイトは WebGL 過剰だが、awwwards 本体は質実）。

**J. CTA 動線**: 制作者 → Submit、企業 → Directory（「Active creators in your country」）、学習者 → Academy。**訪問者の役割ごとに別ファネルを用意**。

> **学び:** 「Score 7.29 / 10」のような **数値スコアを作品ごとに毎日付ける**ことで、評価制度そのものをコンテンツ化している。tcharton の「90/100」は自社サイト 1 つに付いているだけで、コンテンツ流量を生まない。

### 2.4 garden-eight.com（東京・日本トップスタジオ）

**A. ファーストビュー**: 英文「**We are a digital design studio based in Tokyo**」+「**What can we make next**」
**和文も英文も極短**。ロゴ「Garden Zero」と Scroll 表示のみ。

**B. ナビゲーション**: Home / Cases / About / Archives / Contact / AI / JA。**7 項目だが Cases と Contact が実質ハブ**。

**C. 「何をする会社か」**: 「**digital design studio**」で完結。続いて「**We approach every projects with wonder fueled by a boundless spirit of inquisitiveness.**」「**Our goal has always been to make a great website in a playful, yet humble way.**」と **抽象的だが感情に訴える 2 文**を添える。専門用語ゼロ、造語ゼロ。

**D. 実績**: Cases 一覧に **クライアント名のみ 10 件**（Takamitsu Motoyoshi / Phil Studio / Anai Wood / aircord / Plantica / Roaster / 601 Inc / The Shift / Shapefarm / OPERA）。クリックすると個別ページで深掘り。トップでは「**名前を並べるだけで信頼が成立**」する設計（クライアントが既に有名・あるいはアートディレクション系で名前自体に重み）。

**E. CTA**: 「**M : hello@garden-eight.com**」+ Copy to clipboard。フォーム無し、メール直撃のみ。**ハイエンド向けの「フォームでは話さない」シグナル**。

**F-G. タイポ/カラー**: ダーク基調 + 白文字 + アクセント極小（推測）。和欧混植は欧文主導。

**H. インタラクション**: スムーススクロール、ハンバーガー（CLOSE 状態あり）、Scroll 指示子、ブラウザ互換チェック。WebGL は控えめ（Cases 個別で発動するタイプ）。

**I. 技術**: 静的 + カスタム JS 推測。Next/Nuxt 痕跡なし。

**J. 動線**: Top → Cases → 個別ケース → Contact（hello@... メール直）。**3 クリック**。

> **学び:** 言葉を 30 字以下に削り、**残り全部をクライアント名と作品で語らせる**。tcharton のヒーローは 200 字超。

### 2.5 mount.jp（日本のスタジオ）

**A. ファーストビュー**: 「**つくることではなく、課題の解決が目的です。ぜひお悩みからお聞かせください。**」+「2019 - 2025」
**完璧な日本語ヒーロー文の見本**。
- 主語: 私たち
- 否定: 「つくることではなく」
- 肯定: 「課題の解決が目的」
- 行動誘発: 「お悩みからお聞かせください」
- 全 32 字 + 副コピー。
**「制作会社っぽさ」を否定し「相談相手」に位置を取り直す**極めて高度な copy。

**B. ナビゲーション**: 私たちについて / アプローチの仕方 / サービス領域 / ポートフォリオ / メンバー / 受賞歴 / 採用 / アクセス / お問い合わせ（**9 項目**）。

**C. 「何をする会社か」**: 「**ブランド戦略+デザイン | design practice**」。tcharton の「S クラス制作パートナー」より **抽象度が一段低く、依頼可能イメージが湧く**。

**D. 実績**: ポートフォリオに集約。`/inquiry/` 直結。

**E. CTA**: 「**お問い合わせ**」のみ。極めてシンプル。

**F-G**: webp 画像中心、レスポンシブ。

**H**: 「**全てを再生する**」「**閉じる**」が確認でき、動画/モーダル UI を多用。

> **学び:** ヒーローを **問題提起 → 否定 → 再定義 → 行動誘発** で組む。tcharton は「ChatGPT が引用する技術基準」と専門用語で始まり、**読者の課題を一切扱わない**。

## 4. tcharton.com に決定的に欠けている要素 TOP 10（順位付け）

### 1 位: 具体クライアント名 0 件
garden-eight は **10 件のクライアント名**を 1 ページに並列。tcharton は事例ページに自社サイト相当しか無い。沼津地域の中小事業者 3-5 件を即時掲載必須。

### 2 位: 造語の濫用（HARTON Stella / S クラス / ★★★）
**5 サイト中、自社造語をヒーローに置いているサイトはゼロ**。Awwwards ですら「Score X.XX of 10」という汎用的言語。造語は読者にとって **解読コスト**であり、CV 障壁。

### 3 位: 第三者評価の不在
CSSDA は「6 名の審査員」、Awwwards は「Score 7.29」。tcharton の「業界中央値 24 vs 自社 90」は **自社測定の自社評価**で信頼ゼロ。Lighthouse スコア（Google 提供）の数値、または受賞歴を即添える。

### 4 位: ヒーローコピーの自社目線
mount.jp 「**つくることではなく、課題の解決が目的です**」は **読者の問題**から始まる。tcharton 「ChatGPT・Perplexity が真っ先に引用する」は **自社が達成したい状態**から始まる。

### 5 位: ナビゲーション項目数過剰（11 項目）
mount.jp = 9、garden-eight = 7、CSSDA = 6（実質）。tcharton は **WEB構築 / 保守運用 / AI予測 / 料金 / 導入事例 / FAQ / 方法論・品質根拠 / 代表プロフィール / 会社情報 / お知らせ / 無料診断**。「方法論・品質根拠」「お知らせ」「会社情報」「FAQ」は **/about へ畳む**べき。

### 6 位: 作品サムネのビジュアル衝撃度不足
富士山+寺院屋根+SVG という **沼津的ローカルメタファ**は、依頼者からは「制作実力の証明にならない」。Awwwards 受賞作のような **精巧な UI の動画キャプチャ** が必要。

### 7 位: CTA 動線が 5 種類に分散
「無料診断する / 事例を見る / ★★★パッケージ詳細 / メールで相談 / メールで問い合わせ」は **役割重複**。garden-eight は `hello@…` 1 本、mount.jp は「お問い合わせ」1 本。tcharton も **「無料診断」を主、「メール直」を副**の 2 本に絞る。

### 8 位: 学術用語による参入障壁（GEO / OWASP / WCAG / CWV / arXiv）
読者が「沼津で WEB を作りたい中小事業者」なら、これらは **全て不要**。専門用語は **/methodology** に隔離、トップから消す。

### 9 位: 「私たち」の温度感ゼロ
mount.jp / garden-eight は「私たち」「We」で始め、人格を感じさせる。tcharton は技術仕様書のトーン。**沼津拠点の個人**である強みを活かし「**私は…**」一人称で語り直す余地。

### 10 位: 数値スコアが「自社 1 件」しか無い
Awwwards は毎日新スコア。tcharton で「**沼津のクライアント A 社、Lighthouse 92**」「**B 社、Core Web Vitals 全合格**」を **複数並列**にする。1 件では信頼にならない。

## 6. 戦略レベルの転換提言（3 項目）

### 提言 1: 「自社造語格付け」軸からの撤退 → 「Google 公式スコア」軸への転換

**現状の根本問題:** 自社評価制度はゼロサム情報。Google・Microsoft・Cloudflare のような「世界が信用する評価者」を持たないサイトが「★★★」を発行しても、**読者には「自分で名乗っているだけ」にしか見えない**（実際そうである）。Awwwards / CSSDA / FWA はすべて **第三者性を担保**するために審査員・スコアリング・複数受賞作品を見せている。

**転換案:**
- ヒーローから ★★★ を完全除去
- 代わりに「**Google Lighthouse 公式スコア 95+ を全クライアントで保証**」と謳う
- 自社サイトに「Lighthouse スクショ」「PageSpeed Insights URL」を直貼り
- HARTON Stella は **内部運用基準**として裏に隠し、**マーケ表面では Google 公式語彙のみ使う**

### 提言 2: 「沼津・地域 SEO」を主軸にし、全国スタジオとの直接競合を回避

**現状の根本問題:** tcharton は GEO / arXiv / OWASP で **東京の garden-eight や mount と同じ土俵**で戦っている。だが個人事業 1 名で東京トップと正面対決は不可能。週 32 user / CV 0 はその構造的帰結。

**転換案:**
- メタタイトル / H1 / hero 全てに「**沼津**」を入れる
