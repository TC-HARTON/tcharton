## 1. V5→V6 改修内容の効果検証

### 1.1 ヒーロー密度緩和（価格 pills / 信頼バッジ / 業種チップを Trust Strip へ分離）

**Before (V5):** ヒーロー内部に H1 + サブコピー + 数値証跡 + 3 サービスカード + 価格 pills + 信頼バッジ + 業種チップ + メイン CTA + 副 CTA + 公開日付 が同居。情報密度が高すぎ「視線がどこに着地すべきか」が曖昧で、Fitts の法則的にも CTA タップ率を下げるリスクがあった。

**After (V6):** ヒーロー領域は **「H1 → サブコピー → 代表顔 + 数値証跡 → 3 サービスカード → CTA 2 種」** に整理。価格・バッジ・業種は直下の Trust Strip セクションへ移設（index.html L497–544）。

**効果評価:**
- ヒーロー単体の **視覚的階層が明確化** され、最初のスクロールで「主張 → 証拠 → 行動」の F 字導線が成立。
- Trust Strip は border-y で境界を持たせ、独立セクションとして「料金・実績・対応業種」を **横断的に俯瞰させる役割** に昇格。
- 「価格を見る」副 CTA がヒーロー内に残っているため、価格を即知りたいユーザーへの導線も毀損していない。
- スコア寄与: **ヒーロー +1 / バリュープロポジション +1**。

### 1.2 4 ステップ可視化図の新設

**実装:** index.html L546–583。ol 要素で意味的に番号付け、各ステップに **所要時間（30 秒 / 1 営業日 / 30 分 / 3 営業日）を明示**、最後に「ここまで完全無料・断っても可」の安心訴求。

**効果評価:**
- マーケ理論「Conversion Friction の見える化」に正面から応える施策。**「次に何が起こるか分からない」**という最大級のリード離脱要因を全消滅させた。
- 所要時間の数値明示は **「30 秒で始まる」**という心理的軽さを生み、フォーム送信の最終背中押しになる。
- 各ステップに「無料」をベイクしたことで、価格不安をフォーム手前で再洗浄。
- スコア寄与: **コンバージョン動線 +2**。これは V6 最大の効き所。

### 1.3 業種チップのリンク化

**実装:** Trust Strip 内 6 業種すべてが `/services/web/` へのアンカー。hover 時 teal-50 へ色変化。

**効果評価:**
- V5 までは静的テキストで「対応業種を示すだけ」。V6 で **クリック可能な探索 entry point** に進化し、業種別ユーザーの自己同定→詳細ページ遷移という意思決定パスを作った。
- 現状はすべて同一遷移先のため **「業種別 LP」とまでは言えない**。本来は `/services/web/?industry=restaurant` または `/services/web/#industry-restaurant` でセクションスクロールするのが最適解。次フェーズ課題。
- スコア寄与: **コンテンツ深度 +0.5 / コンバージョン動線 +0.5**。

### 1.4 /services/web/ への FAQPage Schema

**実装:** services/web/index.html L77–84。4 質問（プラン選定 / 保守料金 / 制作期間 / 既存サイト改修）。

**効果評価:**
- マーケ視点でも **SERP 上の占有面積拡大 → CTR 改善** という直結効果あり（SEO ではなくマーケ側の便益）。
- 質問 4 が「既存サイト改修できますか?」で **40 万円〜の改修プラン**に言及されており、検索→FAQ リッチリザルト→改修プラン認知という **新しい収益導線** を SERP 段階で作れている。秀逸。
- スコア寄与: **コンテンツ深度 +0.5 / 信頼形成 +0.5**。

## 4. 100 点満点までの残課題（5 点分）

### 4.1 リードキャプチャ強化（残 2 点）
- **メールアドレスのみで取得できる軽量フォーム** を Trust Strip 直下に設置。「12 ページ無料レポート PDF を受け取る」LM 化。
- 現状は /methodology/ 内全文公開で、メアド取得の最小フリクション動線がない。Web3Forms に新エンドポイント追加するだけで実装可能。

### 4.2 業種別深掘り（残 1 点）
- 業種チップが全件同一遷移。`/services/web/#for-restaurant` のように **業種別ペインポイントセクション** を services/web/ に追加し、各チップを別アンカーへ。
- もしくは `/cases/restaurant/` 業種別ケース棚を構築（本質はこちらが上位施策）。

### 4.3 社会的証明の質量強化（残 1 点）
- 「お客様の声 準備中」セクションが正直すぎて **空洞感** がある。代替として **「公開 1 ヶ月での実績数値」**（例: 902 件調査の引用元実数、Mozilla Observatory スコア画面キャプチャ、自社 PageSpeed 100 点スクショ）を即時掲載。
- 自社サイト自体が最大の事例である事実を Visual に翻訳できていない。

### 4.4 Exit Intent / 退出阻止（残 1 点）
- 退出時に「30 秒だけお時間ください: あなたの業種で多い課題 3 つ」のソフトモーダル。reduced-motion 配慮必須。
- 現状すべて静的 CTA のみで、能動的な退出阻止策がゼロ。

## 5. 総評

V6 は **「ヒーロー過密 → セクション分離」** という構造的決断と、**「4 ステップによる動線の透明化」** という心理的決断の二段ロケットで、過去最大級の質的飛躍を遂げた。とくに 4 ステップ図は **「中小企業オーナーが WEB 業者に抱く未知の恐怖」** を真正面から言語化・可視化した、本サイト最高のマーケ装置である。

V6 = **95/100**。S クラスの実装品質に、S クラス相当のマーケ設計が追いついた段階。残 5 点はいずれも追加実装で確実に獲得可能であり、V7 で 100 点到達は現実的。

**V7 推奨優先順位: ① 軽量リードマグネット（PDF メアド取得）→ ② 自社実績の Visual 化 → ③ 業種別アンカー → ④ Exit Intent。**
