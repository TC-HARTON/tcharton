## 2. V6 = 99 の妥当性 — 厳格再評価

**結論: V6 = 99 は過大評価。実態は 91 点。**

V6 の各 +1 点付与は次の前提に依存していた:

- **業種チップ a 化** → 「クリック動機の喚起」（V6 §2.2）。実際には**着地先で業種が出てこない**ため、心理学的には「期待裏切り」を起こし**信頼を毀損**する。+1 ではなく **−1〜0 が妥当**
- **4 ステップ図** → 期間明示 +1 / 始め方 +1（V6 §2.1）。図そのものは valid だが、**ステップ 3 の「30 分」が contact ページで消える** という導線後半の整合性破綻を V6 は検証していない
- **ヒーロー Trust Strip 分離** → 「情報過多解消」（V6 §2.3）。逆に**ヒーロー 1 画面が SVG だけになる**新たな破綻を生んだ。代表肉眼指摘「アニメ動かない / モバイルバランス最悪」の主因はここ

V6 は「コードに <a> が増えた」「<section> が増えた」を加点根拠としたが、**ペルソナがそのリンクを押した先の体験**、**そのセクションが画面に占める割合**を見ていない。机上の構造監査と肉眼の体験監査の乖離が 8 点である。

## 3. 代表指摘 4 点の導線品質への影響評価

### 3.1 「アニメ動かない」

input.css L196–290 / dist/output.css に `@keyframes anim-old-out / anim-particles-fade / anim-new-in / anim-card-up / hero-anim-float` 全件確認。**コード上は動く**。代表指摘の真意は次のいずれか:

- (A) **読み込み遅延** で 0–3 秒の臨界期に間に合わない（最も疑わしい。output.css は dist/ 内で defer 読み込み確認、CSP も問題なし）
- (B) **iOS Safari prefers-reduced-motion = ON** が機種設定でデフォルト ON になっており全アニメ無効化（CSS L368–388 で `animation: none !important`）
- (C) **CSS は動くがアニメ完了後 = SVG が静止画状態を維持** → 代表は「動かない」と表現

(A)(C) のどちらでも結論は同じ: **アニメに依存して 0–3 秒の価値命題を伝えるのは脆弱**。テキスト H1「沼津で『誰も来ない HP』を終わらせる。」が**アニメに対して視覚的劣位**（PC 左右二分割で SVG 480px × text 並列だが、モバイルでは SVG が先に積まれて H1 が下に流れる）。**0–3 秒軸 −3 点**の主因。

### 3.2 「モバイルバランス最悪」

L336: `min-h-screen flex items-center justify-center px-6 lg:px-12 py-24 lg:py-32`。問題点:

1. **min-h-screen + py-24 (= 96px 上下)**: モバイル iPhone 14（844px）では H = 100vh 強制 = SVG 中央寄せで上下に大量の空白
2. **SVG max-w-md = 28rem = 448px**: viewBox 600×600 の正方形 → 高さも約 448px。375px 幅 iPhone でも横はみ出さないが**画面の半分以上を絵が占有**
3. `order-1 lg:order-1` / `order-2 lg:order-2`: モバイル時 SVG 上 / テキスト下の縦積み確定。H1「終わらせる。」は **fold 下** に押し出される
4. ヒーロー直後の Trust Strip（4 価格 pill + 3 バッジ + 6 業種チップ）が flex-wrap で**3 段のチップ墓場**化、視覚密度が反転（ヒーロー疎 → Trust Strip 密）

これがバランス最悪の正体。**モバイル軸 −1**、**0–3 秒軸 −3** の根本原因。

### 3.3 「ボタン最悪」

主 CTA L506–511「30 分 無料診断を受ける」の構造:
- `px-8 sm:px-10 py-4 sm:py-5` / `text-sm sm:text-base lg:text-lg` / `whitespace-nowrap`

問題点:
- **モバイル text-sm + whitespace-nowrap**: 「30 分 無料診断を受ける」+ 矢印 SVG = 約 280px 幅。iPhone 375 - padding 24*2 = 327px に対し**ほぼフィット限界**で**文字が小さく見える**
- 軽 CTA「料金を見る」が同じ高さでなく `py-4`（主 CTA は sm:py-5）→ **横並び時に高さ不揃い**で歪む
- フッター最終 CTA は `bg-dark-900 hover:bg-teal-700`（黒地 → ティール変化）で、ヒーロー主 CTA（teal-700 → teal-800）と**色遷移ロジックが矛盾**。代表の「ボタン最悪」感はこの**一貫性欠如**の生理的不快感

### 3.4 「× 反応しない」

`menu.js` L112–115 で `menuClose` リスナーは存在し、index.html L322 に `<button id="menuClose">` も実在。**コード上は動く**が、コメント L111 に「v1.31 内部閉じるボタン対応（toggle が **inert 化される問題の解消**）」とあり、過去に `setInert(true)` が **menuToggle 自体を inert 化**してしまい × ボタンを別途用意するまで閉じられない事故があった旨が読み取れる。

現状: × ボタンは動く。ただし `setInert` 関数 L53–66 は `document.body.children` のうち `menu` 以外全てに `inert` を付与している。**header（menuToggle 親）も inert 化** → menuToggle 経由では閉じられない / **menu 内の menuClose のみ唯一の閉鎖手段**。代表が「× 反応しない」と感じた瞬間がもし `menuClose` 未実装期だったなら**現在は解決済**。ただし**代表が再度確認していない可能性**があり、信頼回復のため動作デモが必要。
