## 2. 致命的不備（最優先で修正）

### 致命-1: モバイルナビが HTML として壊れている（L162-164）

```html
162:  <a href="/services/web/" ...>WEB構築</a>
163:  <a href="/services/web/" ...                          ← 閉じタグなし、">" もなし、テキストもなし
164:  <a href="/services/ai-prediction/" ...>AI予測</a>
```

L163 は `</a>` も `>` も終端されておらず、次の `<a>` が属性値として吸収される可能性が高い。**spec-checker 上は通っていたとしてもブラウザ DOM 上は確実に破損**。これは ② の sed 一括置換（旧「保守運用」リンク削除）の失敗痕跡。② はここを目視確認していない。

### 致命-2: フッタも同様に壊れている（L378-379）

```html
378: <li><a href="/services/web/" ...>WEB構築</a></li>
379: <li><a href="/services/web/" class="hover:text-white py-3 inline-block"</li>   ← 開きタグの `>` も中身も `</a>` もない
```

致命-1 と同種の sed 修正失敗。`<li>` 内部の `<a>` 要素が未終端で、後続の `<a>AI予測</a>` パースが不定。**フッタは全ページで露出する高頻度領域。これを放置したのは ② の重大な手抜き**。

### 致命-3: 自社引用 blockquote が「3 つ」を明言（L203）

```
「…買い切りで作る、必要なら毎月育てる、予測技術で未来を見る。3 つを切り分けて提供…」
```

代表自身の引用として「3 つ」と書いてある。一方 L261 では「2 軸に切り分け」、L356 では「2 軸（WEB 制作 / AI 予測）」と明言。**同一ページ内で自己矛盾**。これは ② が「3 本柱 → 2 軸」を sed で機械置換した際、blockquote 本文の論理を読まず放置した手抜き。

### 致命-4: 専門領域 dd が「3 要素列挙」のまま（L240）

```
<dd class="text-white">WEB構築 / 保守運用 / AI予測モデル</dd>
```

「保守運用」を独立サービスとして列挙。SPEC v3.6 の「WEB 制作（**保守込み**）+ AI 予測」の 2 軸構造と矛盾する旧 3 要素列挙の残骸。

### 致命-5: ヘッダ PC ナビが index.html の統一基準と不一致（L150）

統一基準（index.html L306）: **`/contact/` → 無料相談**
profile L150: **`/#simulator` → 無料診断**

代表指示で「無料相談に統一」と決めたはずの右端 CTA が、profile では旧仕様「無料診断」のまま。② の PC ナビ修正は profile を含めずに実施した疑い濃厚。

## 4. ② への厳格指摘

② S クラスサイト構築責任者へ。

**結論: profile ページは「修正したつもり」になっている典型的な手抜きの見本である。**

### 指摘 1: sed の盲打ち
モバイルナビ L163・フッタ L379 の **HTML 破損 2 箇所**は、過去の「保守運用リンク削除」を正規表現一括置換で実施し、`</a>` と `</li>` の終端処理を確認せず commit した跡が明白である。spec-checker 上は通ってもブラウザ DOM では壊れる。**目視レビュー義務（SPEC §0.0.7 H-1 disk artifact 確認）を怠った**。

### 指摘 2: 「2 軸」修正の片手落ち
index.html / about / vision で「3 本柱 → 2 軸」を sed 修正したと報告されたが、profile L203 の **代表自身の引用文で「3 つを切り分けて提供」が残存**している。さらに L240 の dd では「WEB構築 / 保守運用 / AI予測モデル」と **3 要素列挙が完全に旧仕様のまま**である。これは sed パターンが単純で本文中の数詞を拾えなかったためで、**ページを読んでいない証拠**。

### 指摘 3: PC ナビ統一の対象外漏れ
PC ナビ右端 CTA を `/contact/` 無料相談に統一する旨の指示があったが、profile L150 は `/#simulator` 無料診断のまま。**全ページ grep を打たずに修正を完了報告した疑い**。SPEC §0.0.3 Failure-Self-Report 違反候補。

### 指摘 4: 内部ジャーゴンの対外露出
L339 の「(SPEC v3.7 §0.0 Ambassadorship Duty 準拠)」は **内部運用ドキュメントの章立てを顧客向けページに露出**させており、明確な手抜き。社外読者には意味不明。L329「2,554 項目」も内部数値で、根拠リンクがないため逆に信頼性を毀損する。

### 指摘 5: 自己矛盾の温存
L261「2 軸に切り分け」L356「2 軸（WEB 制作 / AI 予測）」と書く一方で、L203「3 つを切り分けて提供」L240「3 要素列挙」が同居している。**同一ファイル内で 2 軸と 3 要素が衝突**しており、読者は「結局何軸なのか」を判断できない。これは事業説明として致命的。

### 改善要求

1. 上記 P0 7 件を**即時修正**して spec-checker / verify-all をパスさせよ。
2. P1 5 件は次回 commit で連続対応せよ。
3. 今後の sed 一括置換修正後は **必ず HTML パーサ / linter（または最低限ブラウザ DOM 確認）を経由**せよ。spec-checker.js の 2,554 項目には **未終端 `<a>` を検知する項目が無い** ことが今回露呈した（① への提言として、検証項目追加を推奨）。
4. 「修正完了」報告は `grep -n` 出力 ＋ before/after diff を ① に提示すること。SPEC §0.0.7 H-1 disk artifact 義務の遵守。
