## 4. ② への厳格指摘（SPEC §0.0.3 H-3 観点）

### 4.1 手抜きの根拠（5 点）

1. **HTML 構文エラー 2 件を放置（F1 / F2）**: モバイルナビと footer の `<a>` タグ未閉鎖は、ブラウザが自動補正するため見た目では発覚しにくいが、スクリーンリーダー / SEO クローラー / spec-checker DOM パーサーすべてに影響。**「動いているように見える」だけで合格判定したのは、SPEC §11 の S-RANK 機械保証の精神に反する**。

2. **PC ナビ統一基準の波及確認を放棄（F3）**: 代表は他ページで `想い`（/vision/）を含む 7 リンク統一を進めたが、FAQ モバイルナビは旧 `/about/` `/news/` `/methodology/` 系の残骸を保持。**「PC は直したからモバイルもやったつもり」**という典型的な確認漏れ。複数ページに横展開が必要な変更を 1 ファイルずつ機械的に追わなかった証拠。

3. **AI 予測 Phase 1 期間の自己矛盾を放置（F4）**: FAQ 内部で `4 週 vs 2 週`、サービスページとも食い違う。FAQ 修正時に services/ai-prediction/ を読みに行く義務を怠った。**SPEC §0.0.1 の「整合性確認義務」違反**。

4. **削除済みナラティブ（「Phase 1（無料診断）」混線・F7）**: AI 予測の Phase 用語を WEB 保守 Q24 に流用。事業構造（WEB / AI 2 軸）が頭の中で分離できていない徴候。

5. **③ への報告義務（H-3 / §0.0.7 H-1）違反**: ② は他ページ修正時に「FAQ も連動修正必要」を ① に申告すべきだった。それを怠ったまま push した（疑い）。本監査で発覚するまで自己申告がなかった点こそ、**§0.0.3 H-3 Failure-Self-Report の根本違反**。

### 4.2 構造化データ汚染の指摘（E カテゴリ）

- **行 60-104 の FAQPage schema** は `text` フィールドが本文 DOM と微妙に乖離（schema 行 76 は `/services/web/` リンク無し平文 / DOM 行 256 はリンク有り）。Google Rich Results Test 上は問題ないが、**機械可読性の観点では DOM と schema を完全一致させるのが §11 の哲学**。
