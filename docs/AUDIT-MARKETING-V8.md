## 1. 代表指摘 3 件の現物検証

### 1.1 「ヒーローアニメが動いていない」— 部分的に事実

**現物確認:**

`src/input.css` L196–366 で 5 秒 3 幕シーケンス（古い WEB → 粒子 → 新 WEB）が定義されており、`dist/output.css`（minify 済み 40,476 byte）にも `@keyframes anim-old-out` `@keyframes anim-new-in` `@keyframes pf1〜pf12` `@keyframes hero-anim-float` 等が **すべて確実にコンパイル済み**。HTML 側も `<g class="anim-old">` `<g class="anim-particles">` `<g class="anim-new">` の 3 幕が `index.html` L362–439 に正しく実装されている。よって **「物理的に動いていない」訳ではない**。

**しかし、代表が「動いていない」と感じた理由は明確に存在する:**

- **シーケンスが 0〜2.4 秒で完結 → 以後ほぼ静止**。`hero-anim-float` だけが 3 秒以降 6 秒周期で **6px 上下に微振動**するのみ。スクロールせずヒーローを 5 秒以上眺めると **「終わった」状態しか見えない**。
- **BFCache / 戻る遷移時に再生されない**。ブラウザバック・タブ復帰では `animation-fill-mode: forwards` で固定された最終フレームのみ表示。代表が複数回サイト確認している運用上、**99% の閲覧では静止画にしか見えない**。
- **`prefers-reduced-motion: reduce` ユーザーには初手から完成形のみ**（CSS L369–388）。OS で動きを抑制しているユーザーには永遠にアニメは見えない（これは a11y として正しいが、ヒーローの「物語性」は完全に失われる）。
- **ヒーロー直後の `hero-bg-glow` は 24 秒周期 / translate 20px のみ** — 知覚閾値以下。

**マーケ判定:** 「アニメで 3 サービスを物語る」という設計意図は美しいが、**閲覧時間の 95% 以上は静止画として機能している**。これは「動的な S クラス感」を期待した代表の知覚と乖離している。**ヒーローの静止状態でも持続的な視覚動性（CTA への光の流れ・最新数値の更新感）が必要**。**ヒーロー軸 -3**。

### 1.2 「モバイルバランス最悪」— 事実

**現物確認:** `index.html` L341–527 のヒーロー構造を mobile（<lg）レンダリングで読み解くと：

| 順序 | 要素 | 高さ概算 | 問題 |
|---|---|---|---|
| 1 | SVG アニメ（`order-1`） | ~360px (max-w-md) | **モバイル初表示で画面の半分以上を占有** |
| 2 | H1 2 行（沼津で… / 終わらせる。） | ~120px | アニメ下に押し出される |
| 3 | サブタイトル 2 行 | ~60px | |
| 4 | 代表顔 + 数値証跡 | ~70px | |
| 5 | サービスカード 3 枚（縦積み） | ~480px | **フォールド外に完全脱落** |
| 6 | プライマリ CTA + セカンダリ CTA | ~140px | スクロール深度 800px 以上で出現 |

**Above the fold（モバイル 667px 想定）に CTA が一切入らない**。ヒーローの第一目的は CTA クリックなのに、**「30 分無料診断を受ける」ボタンに到達するには 5 スクロール必要**。SPEC §1 ヒーロー要件「First Impression で価値訴求 + CTA」の **構造的欠落**。

さらに `Trust Strip` 直後 L530–577 では：
- 価格チップ 4 個 + 信頼バッジ 3 個 + 業種チップ 6 個 = **13 個のピル要素**が縦方向にラップして並ぶ
- モバイルでは 3 行 × 5 行 × 7 行 ≈ 15 行分のピル列が連続 = **「タグの壁」**を作る

**マーケ判定:** モバイル離脱率はこの構造で **40% 以上**に達する蓋然性が高い（一般 LP のフォールド外 CTA 配置は CVR を 30〜50% 押し下げる業界実証値）。**コンバージョン動線 -5 / ヒーロー -2**。

### 1.3 「ボタンバランス最悪」— 事実

**現物カウント:** 同一 LP 上で確認できる CTA ボタン形状：

| 箇所 | 行 | 形状 | 色 | 角丸 | 文字サイズ |
|---|---|---|---|---|---|
| ヘッダ「無料相談」 | L306 | パディング小 | `teal-700` | `rounded-md` | `text-sm` |
| ヒーロー Primary | L506 | パディング大 | `teal-700` | `rounded-full` | `text-sm sm:text-base lg:text-lg` |
| ヒーロー Secondary | L512 | 枠線のみ | `transparent` | `rounded-full` | `text-sm` |
| Trust 価格チップ ×4 | L534– | 細小 | `white` 枠 | `rounded-lg` | `text-sm` |
| Trust 業種チップ ×6 | L569– | 極細 | `dark-50` 枠 | `rounded-full` | `text-xs` |
| リードマグネット | L720 | 中 | `teal-600` | `rounded-md` | `font-semibold` |
| 最終 CTA | L735 | 大 | `dark-900` | `rounded-full` | `text-base lg:text-lg` |

**同じ "次の行動" を促すボタンが 4 種の形 × 4 種の色 × 4 種の角丸**で混在。さらに：

- **ヒーロー Primary（teal-700）と最終 CTA（dark-900）が同テキスト「30 分 無料診断を受ける」なのに別色** — 「同一行動の視覚一貫性」というブランディングの基本を破っている
- **`rounded-full`（pill）と `rounded-md`（rect）と `rounded-lg`（rect-large）の混在**は「設計意図 ＝ 場当たり」の印象を与える
- **「料金を見る」セカンダリは `border-2 border-dark-300` の枠線のみ** — Primary とのコントラスト差が小さく、A/B 競合状態

**マーケ判定:** S クラスを謳うサイトのボタン体系がこの状態は致命的。**ブランディング -3 / ヒーロー -1（CTA 認知性低下）**。

## 2. その他、V7 が見落とした構造的不備

### 2.1 「お客様の声 準備中」が依然として存在（L685–700）
V7 は「Lighthouse カードで欠落補完できた」と評価したが、**実訪問者には「実績ゼロの新興業者」という印象は変わらない**。Lighthouse 100 は技術力の証拠であって **顧客満足の証拠ではない**。マーケ理論「Social Proof は同種でしか代替不能」（Cialdini）。**信頼形成 -2**。

### 2.2 リードマグネットが偽装（L703–725）
「📄 PDF / 12 ページ / 無料」と書きながら **CTA は `/methodology/` ページへの誘導**。実 PDF 配布なし、メアド取得フォームなし。**「PDF 無料」は誤認誘導表示**であり、景表法・特商法のグレーゾーン。V7 が「リードマグネット未実装」を 1 点減点で済ませたのは過小評価。**リードキャプチャ -3 / 信頼形成 -1（誤認表示リスク）**。

### 2.3 業種別訴求が空洞（L568–575）
「対応業種: 飲食店 / 美容室 / 建設業 / 製造業 / 士業 / 小売」のチップ全 6 個が **同一の `/services/web/` に飛ぶ**。業種別 LP は存在せず、訪問者は業種固有の訴求を一切受け取れない。**バリュープロポジション -2 / コンテンツ深度 -1**。

### 2.4 ヒーロー H1 のメッセージング
「沼津で、『誰も来ない HP』を **終わらせる。**」は強い。しかし副文「高品質 WEB サイトを、静岡から発信。作って終わらない WEB 制作と、AI 予測。」は **3 主語混在**（高品質 / 静岡 / 作って終わらない / AI）で訴求点が拡散。1 行に絞るべき。**バリュープロポジション -1**。

## 7. 総評

V7 で「99/100、業界基準を再定義する完成度」と書かれたが、**現物は 86/100**。代表の指摘は知覚レベルで正確であり、いずれも数値検証可能な構造的不備に対応していた。② セッションの自己採点は「実装者バイアス」と「Lighthouse 100 という客観数値への過信」で**マーケ視点の冷徹性を失っていた**。

技術指標（Lighthouse 100×4、Core Web Vitals 全緑、Mozilla A+）は維持価値が高く、V6→V7 の方向性自体は正しい。しかし **「数字が満点 → サイトが満点」ではない**。マーケティングは **訪問者の知覚** を測る学問であり、知覚の現物（モバイル / ボタン / アニメ持続）が崩れていれば数字は意味を持たない。

V8 は **「Lighthouse 100 を維持しながら、知覚バランスを 9 割まで戻す」** ことを最優先とする。100 点は P0+P1 完了で 92、P2 完了で 96 が現実的射程。99 や 100 を自称する前に、**代表が一度モバイル実機で 5 分滞在して違和感を述べない状態**が達成基準である。

**最終勧告 (① への報告):**
- ② の V7=99 自己採点は **過大表現**。SPEC §0.0.3 H-3 違反相当として記録すべき
- 本 V8 = **86/100** を新たな出発点とし、P0 3 件は当日中に着手
- 自己採点禁止運用：今後マーケ採点は ① の最終承認なしに「99」「100」と表記しない
