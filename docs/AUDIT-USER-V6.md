## 2. V5→V6 改修の検証結果

### 2.1 4 ステップ可視化図 = 期間明示 +1 / 始め方 +1（合計 +2 点）

`index.html` L547–583 にて新設セクション確認。構造:

```
[1] 無料相談 申込    所要 30 秒
[2] 日程調整 メール   1 営業日以内
[3] 30 分 無料診断   30 分
[4] お見積もり      3 営業日以内
```

**ペルソナ別効果（沼津 50 代飲食オーナー主軸）:**

- 50 代経営者の最大の離脱要因は「**問い合わせたら何が起こるか分からない**」恐怖。4 ステップ図はこれを視覚的に消去
- 「申込 → ハイ営業電話」ではなく「申込 → メール → Zoom → 見積」の **4 段階インターフェース**が見える化
- 末尾「**ここまで完全無料**。納得いかなければ、お断りいただいて結構です。」の安心文言は契約解除コスト不安をゼロ化
- 60 代建設業: ガラケー世代でも「Zoom = 30 分のテレビ電話」と理解可能な単純語彙
- 35 代製造業二代目: 4-5 営業日のスケジュール総和が事業計画に組込可能

**期間明示への波及:**
従来「公開まで何日」が分からなかった。V6 では契約**前**の 4-5 営業日が確定し、契約後の納期は pricing/services 配下に既出のため、**問い合わせ判断の総コストが時間軸で完全に見える化**された。

### 2.2 業種チップ リンク化 = 自分のためのサイト +1 点

`index.html` L536–541 を確認。

```html
<a href="/services/web/" class="text-xs px-3 py-1 bg-dark-50 hover:bg-teal-50 ...">飲食店</a>
<a href="/services/web/" ...>美容室</a>
<a href="/services/web/" ...>建設業</a>
<a href="/services/web/" ...>製造業</a>
<a href="/services/web/" ...>士業</a>
<a href="/services/web/" ...>小売</a>
```

V5 までは `<span>` 要素で**視覚的にリンク風だがクリック不可**だった（V4 以来の継続指摘事項）。V6 で全 6 業種が `<a href="/services/web/">` に置換。

**ペルソナ別効果:**
- 沼津 50 代飲食オーナー: 「飲食店」チップにマウスオーバー → 下線・色変化（hover:bg-teal-50 hover:text-teal-700）→ **クリック動機が視覚的に喚起**
- hover 時に border-color も `hover:border-teal-300` に遷移し、**「押せる UI」の認知科学的シグナル**を完備
- 全業種が同一遷移先（/services/web/）なのは現状の情報設計上適切（業種別 LP は今後の拡張余地）

**残課題:** 業種別アンカー（例: `/services/web/#restaurant`）まで進化すれば追加 0.5 点の余地があるが、配点上限により V6 では満点到達。

### 2.3 ヒーロー視認性改善 = 配点には影響しないが UX 大幅改善

V5 でヒーロー内に同居していた「価格 pill / 信頼バッジ / 業種チップ」を **Trust Strip セクションへ完全分離**（L497–544）。ヒーロー本体は:

- H1 / サブコピー / Lead Evidence（顔写真 + 数値）/ 3 リンクカード / 主 CTA + 軽 CTA / 公開日

の 6 要素に圧縮。**情報過多解消** → 0-3 秒判定の**精度**が上昇（配点は既に満点のためスコア変動なし）。

**情報階層の改善（認知負荷理論）:**
1. ヒーロー = 価値命題（who / what / why）
2. Trust Strip = 価格 + 第三者基準 + 業種適合（filter 軸）
3. 4 ステップ図 = how to start（行動経路）
4. 約束 / 無料レポート / 最終 CTA = 説得 → 行動

スクロール 1 画面ごとに「**役割が単一**」となり、50 代経営者の意思決定モデル（情報を順に処理する逐次型）と完全合致。

### 2.4 /services/web/ FAQPage Schema 追加

`services/web/index.html` L78 で `{"@context":"https://schema.org","@type":"FAQPage",...}` 確認。

**エンドユーザー視点での効果:**
- 直接の UI 変化はゼロ → 100 点採点上の影響はゼロ
- ただし Google 検索結果での **FAQ rich result 表示**により**サイト到達前の信頼形成**を促進する間接効果あり
- これは Marketing / SEO 軸の評価項目であり、本ユーザー軸では**間接波及のみ**としてカウントせず

## 3. V6 残課題（残 1 点 = 信頼軸）

唯一の未達: 信頼軸 14/15。残 1 点は構造的に**時間でしか埋まらない**:

1. **実名顧客の声（最強）** — 公開 1 ヶ月のため案件未完了。最初の納品 + 許諾取得 = 最短 2-3 ヶ月後
2. **代表 30 秒動画** — 顔写真は配置済（L424）。動画は撮影・編集・配置工数で 1-2 週間で実装可能
3. **第三者表彰・認証** — Web Awards 等の応募 → 受賞 = 不確実

**推奨優先度（V7 に向けて）:**

- 短期（即時着手）: 代表 30 秒動画（プロフィールページ → ヒーロー Lead Evidence 内に hover 再生）→ 信頼 14 → 14.5
- 中期（2-3 ヶ月後）: 最初の顧客声 1 件 + 数値（公開後 PV / CV 改善）→ 信頼 14.5 → 15（**満点到達 = 100 点**）
