## 3. V8 で新規発見した 3 摩擦点

### 3.1 リードマグネット PDF 詐称（信頼 −1）

index.html L704–724 の「無料レポート」ブロック:

- L709: H2「静岡県 902 社の WEB 品質調査レポート」
- L713: バッジ「📄 **PDF / 12 ページ / 無料**」（PDF 絵文字 + ページ数明示）
- L720: CTA「レポートを見る →」→ `href="/methodology/"`
- L721: 注釈（極小）「※ 現在は methodology ページで全文公開中」

**ペルソナ反応**: 「PDF / 12 ページ」は強い具体性で**ダウンロード可能なファイル**を期待させる。実際には HTML ページ遷移で、極小注釈 L721 を読まないと気づけない。**ダウンロード型リードマグネット（メアド交換）の擬装に見えるが、実体はただの内部リンク**。SPEC §0.0.1（過大表現禁止）抵触の疑い。

**推奨**: L713 バッジを「📄 Web 公開 / 全 12 章」等に変更し PDF 表記を削除、または PDF を実生成して `/assets/reports/shizuoka-902.pdf` として配布。

### 3.2 contact 主動線構造の Zoom 軽視（始め方 −1）

contact/index.html L201–219:

- L203: 「**推奨**」バッジは **問合せフォーム**に付与（フォームが第一推奨）
- L208–212: フォームブロックがカード化、teal-700 ボーダー 2px、シャドウ強
- L216–218: Zoom 案内は段落テキストで「フォームの『お問い合わせ内容』欄に『ビデオ会議希望』とご記入ください」= **手間 +1 工程**

TOP 主 CTA「30 分 無料診断を受ける」を押した直後の体験として、**「30 分」「Zoom」「無料診断」の言葉が contact 上部に存在しない**。代表 4 ステップ図（L601 ステップ 3「30 分 無料診断 / Zoom で現状把握」）と契約不一致。

**推奨**: contact ヒーローに「30 分 無料 Zoom 診断のお申込み」H1 を追加し、フォーム上部に「☑ Zoom 30 分診断を希望する」チェックボックスをデフォルト ON で配置。

### 3.3 専門用語残存カウント（専門用語 −1）

`grep -c` 結果（index.html）:

| 用語 | 出現箇所 |
|---|---|
| Core Web Vitals | L85（JSON-LD） / L559（Trust Strip 表示） |
| WCAG 2.2 | L86（JSON-LD） / L563（Trust Strip 表示） |
| OWASP Top 10:2025 | L87（JSON-LD のみ・表示なし） |
| Generative Engine Optimization (GEO) | L89（JSON-LD のみ・表示なし） |
| Lighthouse | L648 / L652 / L679（実測カード H2 + 注釈） |
| **総出現数** | **10 件**（うち画面表示は 6 件） |

JSON-LD は機械可読のため不問。**画面表示 6 件**が問題。50 代飲食オーナーは「Core Web Vitals」「WCAG」「Lighthouse」を理解しない。

**推奨**:
- 「Core Web Vitals 全緑」→「表示速度・操作性・安定性すべて Google 合格」
- 「WCAG 2.2 AA 準拠」→「読みやすさ・操作しやすさの国際基準クリア」
- 「Lighthouse Score / 実測値」→「Google 公式 速度テスト 結果」+ 小さく「(Lighthouse v12)」を併記

専門用語の隣に**かならず一般語訳**を置けば −1 を返上できる。
